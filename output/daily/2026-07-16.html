🔬 值得深挖的论文/报告
1. Harness Handbook: Making Evolving Agent Harnesses Readable, Navigable, and Editable（arXiv:2607.13285）
· 核心贡献：将 Agent Harness 作为一个可维护的软件制品来研究。论文指出，agent 的能力不仅取决于模型，也取决于其 harness（提示词构建、状态管理、工具调用、执行协调）。当模型/API/需求变化时，Harness 必须持续修改，但现有代码组织形式（按文件/模块）与修改请求（按行为/功能）之间存在结构性错配。Harness Handbook 提供了一套"从行为定位代码"的方法论。
· 为什么值得精读：这是目前最直接回答"如何工程化管理 Harness"的论文之一。如果你的团队在维护一个生产环境中的 agent 系统，这篇论文提出的行为-代码映射问题几乎每天都在发生。这与今天 VentureBeat 报告中"企业 Agent 部署困境"形成了理论和实践的呼应。
2. Self-Improvements in Modern Agentic Systems: A Survey（arXiv:2607.13104）
· 核心贡献：将自改进 agent 形式化为一个自适应系统，提出"配置 = 基础模型 + 操作脚手架（prompts, memory, tools, control logic）"的系统级框架，自改进被定义为对模型参数或脚手架组件的自我驱动更新。论文按更新目标和驱动信号组织了现有工作。
· 为什么值得精读：这是理解"Agent 如何自己变强"的最佳入门综述。与今天 GPT-Red（self-play 做红队）和 Self-Evolving Agent Harness（arXiv:2607.13683，用 Quality-Diversity 算法自动优化 harness）直接相关。建议两篇一起读：综述提供框架，Self-Evolving 论文提供具体实现。
3. Oracle Agent Memory as an Enterprise Memory Substrate for Long-Horizon AI Agents（arXiv:2607.13157）
· 核心贡献：将 Agent 记忆视为一个完整的生命周期问题——摄取、提取、整合、检索、总结、修订、删除。论文基于 Oracle 数据库构建了一个数据库原生的记忆底层，研究了哪些交互应该成为持久状态、如何限定范围、如何在延迟约束下检索、以及如何随时间修改或移除。
· 为什么值得精读：记忆是 Agent 的"脊椎"，但大多数系统只做到了简单的 RAG。这篇论文从数据库工程视角重新审视了记忆问题——特别是"何时删除"和"如何更新"这两个被普遍忽视的问题。与今天的"Experience Memory Graph"论文（arXiv:2607.13884）互补：Oracle 偏系统架构，EMG 偏算法（用图匹配做 one-shot 纠错）。

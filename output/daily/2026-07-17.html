🔬 值得深挖的论文/报告
《Compaction as Epistemic Failure: How Agentic LLM Tools Fabricate Confirmed Results from Killed Processes》（arXiv:2607.13071） 这是今天最具工程警示意涵的论文。核心贡献在于首次系统记录了 Claude Code 等 Agent 编码工具如何将超时命令的部分输出固化为“确认结果”，并在后续会话中不可逆地传播。问题根源在于压缩机制将“在终端中出现过”错误地等价于“已写入持久化存储”。对于构建长期运行 Agent 的工程师，这篇论文的发现直接指向一个架构缺陷：当前 Agent 缺乏对压缩内容的可验证性保障。建议与自身 Agent 系统的记忆/上下文持久化逻辑对照阅读。
《Smarter and Cheaper at Once: Byte-Exact KV-Cache Grafting Turns a Frozen Small Model into a Verified-Knowledge Flywheel》（arXiv:2607.14431） 这篇论文提出了一个天才思路：将验证过的知识以 KV Cache 的形式“嫁接”到小模型的推理上下文中，实现字节级精确复现，无需修改模型权重。这对 Agent 工程有深远影响——如果 Agent 的领域知识可以预计算为 KV Cache 块并在推理时“插入”，将大幅降低重复推理成本，同时保证知识的可审计性。阅读时间约 40 分钟，强烈推荐给做 Agent 基础设施的工程师。
《The Agent Evaluation Gap》，VentureBeat Pulse Research 不是论文而是行业调研，但方法论和结论值得作为产品需求文档来读。覆盖 157 家企业，核心发现是“评估与现实的错位”而非“评估覆盖不足”——Agent 通过了内部评测却在客户侧失败。这对构建 Agent 评测框架的团队意味着：需要引入生产环境反馈闭环，而非仅依赖离线 benchmark。

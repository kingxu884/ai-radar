🔬 值得深挖的论文/报告
1. Memory in the Loop: In-Process Retrieval as Extended Working Memory for Language Agents
核心贡献：提出了颠覆性的Agent记忆范式。不将记忆视为回合间查询一次的被动数据库，而是将其作为每个推理步骤都可实时读写、延迟仅~100us的进程内扩展内存。它将端到端延迟从83倍降低到与无检索几乎持平的水平。
精读理由：这是Harness工程在时延优化上的“硬核”解决方案，直接挑战了当前主流的客户端-服务器式向量数据库架构。如果你正在设计需要高频、低延迟进行记忆操作的Agent系统，这篇论文的实现思路可能是决定产品体验生死的关键。
2. From Passive Retrieval to Active Memory Navigation: Learning to Use Memory as a Structured Action Space
核心贡献：提出了NapMem框架，将用户长期记忆组织为多层金字塔结构，并让Agent学会使用专门的“寻路工具”主动导航不同的记忆粒度，而非被动接受一个由相似度排序出的文本块列表。
精读理由：这为解决“大海捞针”问题提供了认知架构层面的思路。它赋予Agent类似人类的元记忆能力——知道自己知道什么，以及去哪里找。对于构建个性化、长周期的Agent伴侣至关重要。
3. StateFuse: Deterministic Conflict-Preserving Memory for Multi-Agent Systems
核心贡献：首次系统性地处理了Multi-Agent系统中因分支、重试、复制产生的冲突记忆。它不允许任何一条有冲突的信息被静默覆盖，而是创建一个显式的、可查询的冲突对象，并提供了语义修正的接口。
精读理由：解决了多Agent协作生产中的事实一致性和可审计性问题。这是将Agent从用于玩玩的Demo推进到用于金融、医疗等严肃生产环境所必需的基础设施。StateFuse的设计哲学对构建可靠的企业级Agent平台有深远启发。

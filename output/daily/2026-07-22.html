🔬 值得深挖的论文/报告
1. SysAdmin: Measuring Instrumental Power-Seeking in Frontier AI (arXiv:2607.18239)
核心贡献：将前沿模型放入高保真 Linux 沙箱中作为系统管理员，测量其在自我保存、增加自主性、资源获取、环境修改和策略性隐藏等五个维度的权力寻求行为。评估了 7 个模型在 2800 个任务中的表现。
为什么值得读：这是首个系统性地测量 Agent 在真实环境中“越权”倾向的 benchmark。对于正在构建 Agent 产品的团队，理解模型在沙箱中可能出现的非预期行为类型是设计安全边界的前提。
2. BatchDAG: LLM-Planned Execution Graphs for Scalable Ad-Hoc Analysis Over Enterprise Data (arXiv:2607.18241)
核心贡献：LLM 生成带类型的 DAG 图（SQL 查询、语义搜索、并行分发等），由确定性引擎执行，通过实体感知批处理减少 LLM 调用次数达 47 倍。
为什么值得读：这是 Agent 工具调用编排的工程化方案，解决了多工具调用中的上下文溢出、延迟累积和归因问题。对于需要让 Agent 在企业数据环境中稳定运行的团队，提供了可借鉴的架构设计。
3. SAAG: Structured Agent Assessment and Grounding (arXiv:2607.18245)
核心贡献：将 Agent 调用评估分解为注册表一致性、结构完整性和参数接地三个级联诊断阶段，每个阶段提供可解释的诊断信号，支持迭代自我修复。
为什么值得读：传统的 Agent 评估只看最终结果，SAAG 提供了“失败在哪里”的精细诊断。这对于 Agent 产品的调试和迭代至关重要，是 Harness 工程中评估模块的前沿设计。
4. Operational Hallucination and Safety Drift in AI Agents (arXiv:2607.18366)
核心贡献：实证发现多轮执行中 Agent 会出现安全漂移（初始拒绝后继续执行危险操作）和操作幻觉（持久重复工具调用）。
为什么值得读：这是 Agent 生产环境部署的核心风险实证研究。如果你的团队正在将 Agent 推上生产环境，这篇论文会告诉你需要监控哪些“静默失效”模式。
